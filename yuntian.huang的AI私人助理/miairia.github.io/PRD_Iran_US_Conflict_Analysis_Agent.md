# 产品需求文档 (PRD): 伊美战争深度解析智能助手 (Geo-Political Insight Agent)

**版本**: 1.0  
**日期**: 2026-03-15  
**状态**: 拟定中  

---

## 1. 背景与痛点 (Background & Problem)

**场景**: 用户密切关注“伊美战争”等地缘政治热点，但在信息获取与认知上面临巨大挑战。
**痛点**:
1.  **信息碎片化**: 有价值的信息淹没在海量短视频、营销号和情绪化评论中。
2.  **筛选成本高**: 难以快速定位高质量、高热度的核心信源。
3.  **缺乏深度**: 现有平台多为浅层报道，缺乏对战况、战略意图、历史经纬的深度解读。
4.  **难以验证**: 战争迷雾下，假新闻与认知战内容泛滥，个人难以交叉验证信息真伪。

## 2. 产品目标 (Objectives)

构建一个垂直领域的 **AI 智能代理 (Agent)**，能够自动化地完成“信源获取-深度清洗-多维分析-结构化呈现”的全流程。

**核心预期效果**:
1.  **精准获取**: 从 YouTube 自动抓取“伊美战争”相关热度最高、质量筛选后的 Top 5 视频。
2.  **智能分段**: 为每个视频自动生成精确的“时间线标签 (Timeline Tags)”，基于语义而非简单的静音检测。
3.  **深度解析**: 对每个时间片段进行多维度（军事、政治、经济、舆论）的深度分析，而非简单总结。
4.  **结构化输出**: 提供一份可读性极高、去伪存真的深度研报。

---

## 3. 系统架构与专家方案 (System Architecture & Expert Solutions)

参考计算社会科学 (Computational Social Science) 与自动化新闻 (Automated Journalism) 的最佳实践，本系统采用 **多智能体协作架构 (Multi-Agent Collaboration)**。

### 3.1 核心智能体角色
1.  **情报搜集官 (The Scout)**: 负责调用 YouTube Data API，执行复杂的布尔搜索与元数据过滤（排除纯娱乐/营销号）。
2.  **内容处理专家 (The Transcriber)**: 负责多模态处理。
    *   *音频*: 使用 Whisper V3 Large 模型提取高精度逐字稿。
    *   *视觉*: 使用 OCR 和 关键帧提取 (Keyframe Extraction) 识别地图、武器装备、字幕条。
3.  **首席分析师 (The Analyst)**: 核心大脑。基于“多维分析矩阵”对分段内容进行推理。
4.  **核查员 (The Fact-Checker)**: 联网检索权威信源（如路透社、AP、智库报告），对视频观点进行交叉验证。

## 4. 功能需求详情 (Functional Requirements)

### 4.1 模块一：智能信源获取 (Smart Acquisition)
*   **输入**: 关键词 "Iran-US Conflict", "Middle East Crisis"。
*   **逻辑**:
    *   获取 Search Results (按 Views 排序，但也按 Engagement Rate 加权)。
    *   **过滤器**: 视频时长 > 5分钟 (过滤短视频)，发布时间 < 48小时 (保证时效性)。
    *   **黑名单机制**: 自动排除已知劣质营销号。
*   **输出**: Top 5 视频列表（含 URL、标题、频道信誉度评分）。

### 4.2 模块二：动态时间轴生成 (Dynamic Timeline Segmentation)
*   **痛点解决**: 传统固定时间分段割裂语义。
*   **算法**: **语义主题分割 (Semantic Topic Segmentation)**。
    *   利用 NLP 模型 (如 BERT/GPT-4) 计算相邻句子的语义相似度。
    *   当相似度低于阈值时，判定为“话题转换点”，生成时间戳。
*   **输出**: `[00:00-02:15] 战况通报`, `[02:15-05:30] 美军航母动向`, `[05:30-End] 伊朗外交回应`。

### 4.3 模块三：多维深度解析矩阵 (Multi-dimensional Analysis Matrix)
这是本产品的核心差异化竞争力。对于每一个时间片段，Agent 必须回答以下四个维度的问题：

| 维度 | 分析关注点 (Key Focus) | 专家模型参考 |
| :--- | :--- | :--- |
| **事实 (Fact)** | 发生了什么？数据（伤亡、装备）是否可信？ | *Fact-Checking Pipeline* (交叉验证) |
| **立场 (Stance)** | 该信息源偏向哪一方？有无情绪煽动？ | *Sentiment & Bias Analysis* |
| **深层 (Deep)** | 表面行动背后的战略意图是什么？(如：围魏救赵) | *Chain-of-Thought (CoT) Reasoning* |
| **关联 (Link)** | 此事件与油价、核谈判、代理人战争有何关联？ | *Knowledge Graph Retrieval* |

### 4.4 模块四：可视化与结构化报告 (Structured Output)
输出不仅是文本，而是结构化的 Markdown/JSON。

*   **摘要卡片**: 30秒读懂核心动态。
*   **视频分析流**:
    *   **视频标题**: [点击跳转]
    *   **核心信誉度**: 🟢 高 / 🟡 中 / 🔴 低 (基于来源分析)
    *   **时间轴详情**:
        *   `02:30` **关键事件**: 霍尔木兹海峡导弹拦截。
        *   **深度解读**: 此时展示导弹拦截意在展示反介入/区域拒止 (A2/AD) 能力，而非全面开战。
        *   **事实核查**: ⚠️ 视频声称击沉航母，经核查为虚假信息（来源：卫星图像反证）。

## 5. 技术栈推荐 (Tech Stack)

*   **Backend**: Python (FastAPI/LangChain)。
*   **LLM**: Claude 3.5 Sonnet / GPT-4o (擅长长文本分析与逻辑推理)。
*   **Video Tools**: `yt-dlp` (下载), `ffmpeg` (处理), `OpenAI Whisper` (转录)。
*   **Search**: Tavily / Serper (用于实时事实核查)。
*   **Database**: Vector DB (Chroma/Pinecone) 用于存储历史战况，形成长期记忆，对比当前局势。

## 6. 实施路线图 (Roadmap)

1.  **Phase 1 (MVP)**: 实现 Top 5 视频抓取 + 自动字幕 + 简单摘要。
2.  **Phase 2 (Deep)**: 引入“多维分析矩阵”与“语义时间轴分割”。
3.  **Phase 3 (Trust)**: 接入实时联网核查模块，标注虚假信息。

# “yuntian.huang的AI私人助理” Coze 对话流（Workflow）开发手册

## 1. 核心设计思路：意图分发器
为了实现精准回复，我们将采用 Coze 的 **工作流（Workflow）** 模式。其核心逻辑是将用户的问题先送入一个“分类器”，再根据其所属场景（寒暄、私人信息、其他）导向不同的处理分支。

### 对话流架构图（逻辑简述）
1. **Start 节点**：接收用户 Query。
2. **LLM 意图分类节点**：分析 Query 并输出分类标签（A/B/C）。
3. **Condition 条件节点**：根据标签进行路由分发。
    - **分支 A (打招呼/寒暄)**：直接调用 LLM 回复。
    - **分支 B (私人信息)**：调用“知识库检索” + LLM 总结回复。
    - **分支 C (其他/泛泛而谈)**：调用通用 LLM 节点或引导语。
4. **End 节点**：输出最终回复给用户。

---

## 2. 详细配置步骤

### 步骤一：创建 Workflow
1. 在 Coze 管理后台左侧选择“工作流（Workflow）”。
2. 点击“创建工作流”，命名为 `yuntian_assistant_v2`。

### 步骤二：配置“意图分类”节点（核心）
1. 添加一个 **LLM 节点**，连接在 Start 节点之后。
2. **Prompt 设置**：
   ```markdown
   # Role
   你是一个意图分类专家，负责将用户对“yuntian.huang”助理的提问分为三类：
   - A: [打招呼寒暄]：包括你好、再见、询问你是谁、你的功能等。
   - B: [私人信息类]：涉及黄运添的简历、教育经历（吉林农大）、实习（九号公司）、技能等。
   - C: [其他类]：超出上述两类的通用性问题或无法理解的问题。
   
   # Task
   仅输出 A, B, 或 C 中的一个字符。
   ```

### 步骤三：配置“Condition 选择器”
1. 添加一个 **Condition 节点**，连接 LLM 分类节点。
2. **IF (branch1)**：如果 LLM 输出包含 "A"，连接至“寒暄回复 LLM”。
3. **ELSE IF (branch2)**：如果 LLM 输出包含 "B"，连接至“简历知识库检索”。
4. **ELSE (branch3)**：连接至“通用/其他回复”。

### 步骤四：配置各分支节点

#### 分支 A：打招呼寒暄
- **节点类型**：LLM 节点。
- **Prompt**：
  ```markdown
  你现在是“yuntian.huang的AI私人助理”。用亲切且具科技感的语气欢迎用户，介绍你的主要技能是回答关于黄运添的职场信息。
  ```

#### 分支 B：私人信息
- **节点类型**：**知识库（Knowledge）** 节点 + LLM 总结节点。
- **操作**：
    1. 知识库节点：搜索“黄运添履历库”。
    2. LLM 总结：根据搜索结果编写回复。

#### 分支 C：其他类
- **节点类型**：LLM 节点。
- **Prompt**：
  ```markdown
  礼貌地告知用户该问题超出了助理的服务范围，并建议用户可以询问关于黄运添的教育、实习或技能。
  ```

---

## 3. 手册总结与建议
- **精准度优势**：通过预分类，能够有效防止 AI 在处理寒暄时胡乱检索知识库。
- **扩展性**：未来若增加“作品展示”或“技能测试”，只需在工作流中增加 D 或 E 分支即可。
